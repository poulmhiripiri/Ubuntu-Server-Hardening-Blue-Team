# Author Background

This project was developed by **Poul Mhiripiri**, an IT Networks and Infrastructure professional with over 15 years of hands-on experience across ISP, banking, enterprise infrastructure and cybersecurity environments.

## ISP Linux, DNS, Mail and Hosting Experience

My Linux server experience began in the ISP sector, where I worked on production systems supporting customer-facing internet services. This included setting up client domains on ISP authoritative name servers, configuring and maintaining Linux-based DNS services using BIND, and supporting hosted customer domains.

I worked with Linux-based mail and proxy services deployed at client sites, including environments where DNS, mail routing, internet access and perimeter services were critical to daily operations. As part of ISP hosting operations, I maintained MX servers and hosted web servers used by customer domains. This included spam filtering, mail relay support, DNS records, hosted web services and availability checks.

## Banking and Enterprise Infrastructure Experience

In banking and enterprise infrastructure roles, I extended this experience into security-focused and highly available environments where uptime, patching, access control, firewall policy, logging, audit evidence and infrastructure resilience were essential.

This project brings together that practical infrastructure background with my current focus on cybersecurity and blue-team defensive operations.

## Why this project is relevant

The purpose of this repository is to demonstrate practical Linux server hardening skills, security posture assessment, attack surface reduction and evidence-based infrastructure security using tools such as Lynis, UFW, Fail2Ban, SSH hardening, sysctl tuning and security logging.

It shows how hands-on infrastructure experience can be translated into blue-team security outcomes: prevention, detection readiness, operational resilience, audit evidence and secure baseline configuration.
